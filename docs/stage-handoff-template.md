# 阶段交接模板

## 目标
- 为阶段切换提供统一、清晰的交接格式。
- 降低多 Agent 协作时上下文丢失、重复判断和误解风险。
- 让下一阶段能快速判断：是否可继续、该读什么、还缺什么。

## 适用范围
- 适用于任意阶段结束后，准备进入下一个阶段时的交接说明。
- 特别适用于：
  - Stage 0 -> Stage 1
  - Stage 1 / 1.5 -> Stage 2
  - Stage 2 -> Stage 3
  - Stage 3 -> Stage 4
  - Stage 4 -> Stage 5

## 使用规则
- 发生阶段切换时，输出中应包含本模板的核心内容。
- 若当前阶段未完全满足退出条件，也应明确说明：
  - 哪些条件已满足
  - 哪些条件未满足
  - 为什么仍建议继续或暂缓继续

## 推荐交接格式
```md
1. 当前阶段
- 当前阶段名称

2. 交接结论
- 可进入下一阶段 / 暂不建议进入下一阶段

3. 已完成内容
- 当前阶段已经完成的关键产出

4. 已更新文档
- 本阶段新增或更新的文档

5. 下一阶段必读输入
- 建议下一阶段优先阅读的文档和上下文

6. 未解决问题
- 仍待确认的问题

7. 假设与风险
- 当前采用的关键假设
- 若继续推进可能带来的风险

8. 建议下一角色
- 推荐进入的角色或阶段
```

## 交接检查项
### 通用检查
- 是否明确说明当前阶段是否达到退出条件
- 是否列出下一阶段所需输入
- 是否列出未解决问题和风险
- 是否指出哪些文档已更新

### 实现前交接
- 是否已明确任务 ID、输入、输出、依赖和完成定义
- 是否已确认实现所需文档齐备

### 评审前交接
- 是否已明确改动范围
- 是否已说明测试结果与未覆盖项
- 是否已检查文档同步需求

## 备注
- 阶段交接模板用于提高协作效率，不要求每次写很长。
- 关键是让下一阶段“不需要重新猜一次当前状态”。
