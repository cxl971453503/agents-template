# 测试策略

## 测试目标
- 确保核心链路可用
- 降低回归风险
- 保证任务、项目、评论等关键模块的基本正确性

## 测试层次
### 单元测试
覆盖：
- 状态流转逻辑
- 输入校验
- 服务层纯业务逻辑

### 集成测试
覆盖：
- API 输入输出
- 数据库读写流程
- 权限边界（MVP 下为简化版本）

### E2E 冒烟测试
覆盖：
- 登录
- 创建项目
- 创建任务
- 更新任务状态
- 添加评论

## 优先级
1. 认证
2. 任务增删改查
3. 评论流程
4. 项目列表

## 发布门槛
发布前至少满足：
- 所有关键 API 测试通过
- 至少 1 条核心 E2E 路径通过
- 无阻塞级 bug
