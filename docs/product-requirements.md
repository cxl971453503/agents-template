# Product Requirements

## Problem Statement
当前小团队常将任务信息散落在聊天工具、文档和表格中，导致任务状态不透明、责任不清晰、跟进成本高。

## Users and Scenarios
### User A: 项目负责人
需要快速查看项目当前有哪些任务阻塞，谁负责，何时应完成。

### User B: 执行成员
需要清楚看到分配给自己的任务、优先级和上下文评论。

## MVP Features
1. 用户注册与登录
2. 创建工作区
3. 创建项目
4. 创建任务
5. 编辑任务标题、描述、状态、负责人、截止日期
6. 评论任务
7. 按状态和负责人筛选任务

## User Stories
- 作为项目负责人，我希望创建项目并录入任务，以便组织团队工作。
- 作为团队成员，我希望只看与我有关的任务，以便快速处理。
- 作为协作者，我希望在任务下评论，以便补充上下文。

## Acceptance Criteria
### 登录
- 用户可以通过邮箱和密码登录
- 登录失败时有明确错误提示

### 任务创建
- 用户可以输入标题和描述创建任务
- 任务必须属于某个项目
- 创建成功后在任务列表可见

### 任务指派
- 用户可以选择工作区中的成员作为负责人
- 负责人信息在任务详情和列表中可见

### 评论
- 已登录用户可以在任务详情页发表评论
- 评论按时间顺序展示

## Non-goals
- 不支持子任务
- 不支持附件上传
- 不支持复杂自动化流程

## Open Questions
- MVP 是否需要邀请成员功能
- 是否需要任务优先级字段
- 是否支持公开分享项目链接
