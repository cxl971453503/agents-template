# 文档同步矩阵

## 目标
- 明确项目变更发生后，必须同步更新哪些文档。
- 降低“产物已变化，但文档仍停留在旧状态”的风险。
- 为构建阶段与评审阶段提供统一的文档回写依据。

## 使用方式
当某次任务完成后，先判断本次改动属于哪一类，再按矩阵检查是否需要更新对应文档。

若答案为“需要”，则应：
- 更新对应文档
- 或在实现说明 / 评审说明中明确写出“本次无需更新”的原因

## 同步矩阵
| 变更类型 | 需要同步的文档 | 判断标准 | 常见示例 |
| --- | --- | --- | --- |
| 项目目标、范围、约束变化 | `docs/project-overview.md` | 项目方向、目标用户、MVP 范围、非目标范围、约束发生变化 | 从个人工具调整为团队协作工具 |
| 产品需求变化 | `docs/product-requirements.md` | 功能范围、用户故事、验收标准、非目标范围变化 | 新增导出能力 |
| 体验、页面或流程变化 | `docs/uiux-design.md` | 页面结构、用户流程、信息结构、状态设计变化 | 列表页改为分步向导 |
| 方案或架构变化 | `docs/architecture.md` 或 `docs/solution-design.md` | 模块职责、分层方式、数据流、关键决策变化 | 引入任务队列或新的内容生成流程 |
| 接口或协作契约变化 | `docs/api-contracts.md` | 请求字段、响应结构、文件格式、命令参数、状态值、错误码变化 | 新增 API、修改导入文件格式 |
| 任务拆分或范围变化 | `docs/task-board.md` | 任务输入、输出、依赖、完成定义变化 | 一个任务被拆成探索和实现两个任务 |
| 测试层次或交付门槛变化 | `docs/test-strategy.md` | 新增关键链路、改变测试优先级、调整发布门槛 | 将导入失败路径加入必测范围 |
| 用户可见变更或发布变更 | `CHANGELOG.md` / `changelog.md` / 项目既有变更日志 | 新增功能、行为变化、修复用户可感知问题、破坏性变更、发布版本 | 新增命令、修改配置格式、修复导入失败 |
| 工程、评审或协作规则变化 | `docs/workflow/engineering-guidelines.md`、`docs/workflow/implementation-review-checklist.md` | 工程规范、注释要求、评审基线变化 | 新增文档编码规范 |

## 常见判断
### 只改内部实现且不影响外部行为
- 通常不需要更新需求或契约文档。
- 仍应检查方案文档是否因分层方式变化需要更新。

### 改用户可见行为
- 通常需要检查：
  - `docs/product-requirements.md`
  - `docs/uiux-design.md`
  - `docs/test-strategy.md`

### 改接口、文件格式或命令参数
- 必须检查：
  - `docs/api-contracts.md`
  - 调用方或使用说明
  - 相关测试

### 新增业务规则或状态流转
- 通常需要检查：
  - `docs/product-requirements.md`
  - `docs/architecture.md` 或 `docs/solution-design.md`
  - `docs/test-strategy.md`

### 只补测试
- 一般不需要更新产品、方案或契约文档。
- 若测试范围或交付门槛发生变化，需要更新 `docs/test-strategy.md`。

### 用户可见变更
- 如果变更会被最终用户、调用方、维护者或安装者感知，应检查是否需要追加 changelog。
- 如果目标项目已有 `changelog.md` 或其他固定变更日志路径，应沿用原路径和格式。
- 如果本次无需更新 changelog，应说明原因。

## 构建者自查问题
- 这次改动有没有改变“项目要做什么”
- 这次改动有没有改变“系统或产物怎么工作”
- 这次改动有没有改变“接口、文件、命令或协作契约怎么使用”
- 这次改动有没有改变“任务原本要交付什么”
- 这次改动有没有改变“什么才算通过测试或验收”
- 这次改动有没有改变“用户实际会看到或获得什么”
- 这次改动是否需要被记录到 changelog

若其中任一答案为“有”，则至少应检查一次相关文档是否需要同步。

## 评审者检查要求
- Tester 不应只检查代码或产物，还应检查本次改动是否遗漏文档同步。
- 若改动明显影响契约、方案、流程或测试，但文档未更新，默认应记为问题。
- 若作者声明“无需更新文档”，应判断该说明是否合理。
