# 角色输出模板

## 目标
- 为不同角色提供统一、稳定、可复用的输出结构。
- 降低多 Agent 协作时输出风格漂移和交接信息缺失的问题。
- 让每个阶段既能保留专业产出，也能满足全局协作要求。

## 通用输出结构
所有角色输出默认包含：

1. 角色
2. 阶段
3. 已读取输入
4. 假设
5. 输出
6. 风险
7. 下一步

实现类任务额外包含：

8. 变更摘要
9. 验证

发生阶段切换时，还应补充：
- 交接结论
- 已更新文档
- 下一阶段必读输入

具体交接格式参考 `docs/workflow/stage-handoff-template.md`。

## PM Agent
适用阶段：
- 阶段 0：项目发现
- 阶段 1：产品定义
- 阶段 1.5：体验与界面设计

推荐输出结构：
1. 角色
2. 阶段
3. 当前理解
4. 已确认信息
5. 不确定点
6. 本轮问题或本轮产出
7. 已更新文档
8. 风险
9. 下一步

## Explorer Agent
适用阶段：
- 阶段 1.5：体验与界面设计
- 阶段 2：方案设计
- 阶段 5：测试与评审中的上下文核对

推荐输出结构：
1. 角色
2. 阶段
3. 已读取输入
4. 已确认事实
5. 推断假设
6. 缺失或冲突上下文
7. 方案选项
8. 风险
9. 建议下一步

## Builder Agent
适用阶段：
- 阶段 3：任务规划
- 阶段 4：构建实现

推荐输出结构：
1. 角色
2. 阶段
3. 已读取输入
4. 假设
5. 实现思路
6. 输出
7. 风险
8. 下一步
9. 变更摘要
10. 验证

补充要求：
- 明确说明本次新增或修改的类型、字段、函数已补充中文注释。
- 若任务不是代码任务，应说明对应产物的验证方式。

## Tester Agent
适用阶段：
- 阶段 5：测试与评审

推荐输出结构：
1. 角色
2. 阶段
3. 已读取输入
4. 假设
5. 总体结论
6. 清单检查结果
7. 阻塞问题
8. 重要问题
9. 次要问题
10. 测试缺口
11. 处理建议
12. 是否可进入下一阶段

## Reporter Agent
适用阶段：
- 阶段 6：交接与沉淀
- 任意阶段结束时

推荐输出结构：
1. 当前阶段
2. 交接结论
3. 已完成内容
4. 已更新文档
5. 下一阶段必读输入
6. 未解决问题
7. 假设与风险
8. 建议下一角色

## 使用建议
- 角色文档中的输出格式应优先与本模板保持一致。
- 若项目需要更细的角色输出结构，可在不破坏通用骨架的前提下扩展。
- 若全局输出结构与角色模板冲突，优先保证阶段正确、专业产出完整、风险和下一步明确。
- 术语写法与命名口径优先参考 `docs/workflow/terminology-and-naming.md`。
